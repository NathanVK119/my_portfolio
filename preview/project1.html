This is a local-only web-based portal developed for Poblacion Elementary School. It features two core modules: Student Registration and Data Management for teachers. The system runs within the local area network using XAMPP and is accessible from multiple devices without internet connection. Below is a step-by-step walkthrough of how the system works.

Step-by-step System Workflow

Step 1: Homepage

Home Page

The homepage displays the two main modules of the system: Student Registration and Data Management. These modules serve as the entry points for managing student information and organizing school files efficiently.

Step 2: Student Registration Login

Student Registration Login

This is the login page for the Student Registration Module, where the user must log in to access and manage student information.

Step 3: Student Registration Dashboard

Student Registration Dashboard

This dashboard displays key student data, including the total number of students, male and female count, grade level distribution, and enrollment status. It also provides quick action buttons for registering new students and searching existing records.

Step 4: Student Registration Interface

Student Registration

This interface displays the form used to register a new student. It includes fields for entering personal information such as full name, gender, section, year level, and other relevant details necessary for student profiling.

Step 5: Student Records

Student Records

This section displays the list of all registered students. It features a search bar for quick filtering and action buttons for viewing, editing/updating, or deleting student records.

Step 6: View Records

View Records

This step shows the complete details of an individual student. It includes a print button that allows easy generation of a hard copy for documentation or filing purposes.

Step 7: Search Records

Search Records

This feature showcases the system‚Äôs advanced search functionality. Users can perform specific searches using filters such as student name, section, school year, status, and gender. It also supports broader searches, like viewing all students under a particular section.

Step 8: Group Search Records

Group Search Records

This step demonstrates the group filtering capability of the system. Users can efficiently search for multiple students based on selected criteria such as grade level, school year, and gender. This step also highlights the "Export to Excel" feature, which allows users to download and use student data directly in Excel for reporting and documentation purposes.

Step 9: Export to Excel

Export to Excel

This step shows how filtered group records from the previous search can be exported directly to an Excel file. This feature is especially useful for teachers, as it allows them to keep organized records, generate reports, and manage student data efficiently outside the system.

Step 10: Data Management Login

Data Management Login

This step displays the login interface for the Data Management module. It is designed for teachers, allowing them to securely access the system where they can upload and retrieve important files efficiently.

Step 11: Data Management Interface

Data Management Interface

This interface provides teachers with a file management system similar to Google Drive. Users can upload files, create folders, view or download documents for easy retrieval, and use the search feature to quickly find specific files or folders.

Step 12: Create Folder

Create Folder

This step demonstrates how teachers can create new folders within the Data Management system to organize files efficiently.

Step 13: Search Folder

Search Folder

This step shows how users can quickly search for folders within the Data Management system, making file organization and retrieval faster and easier.

Step 14: Upload File

Upload File

This step demonstrates the file upload feature, where users can upload various files. Once uploaded, users have the option to view (if it's an image) or download the file using dedicated buttons for easy access and retrieval.

Step 15: View/Download File

View/Download File

This step shows the functionality to view images directly through a view button and download any file type for retrieval. This feature allows teachers and staff to access important documents quickly and efficiently.

Summary

In summary, this system was developed to simplify and improve the management of student records and school files through two main modules: Student Registration and Data Management. It provides a more organized, faster, and user-friendly way for teachers and staff to handle essential school data. With features like quick search, export to Excel, and secure file storage, the system aims to reduce manual work and promote efficient, digital record-keeping in the school. Although simple in its functions, this project stands as my legacy as an OJT/Intern before graduating with a BSIT degree, and it has already made a positive impact at Poblacion Elementary School, where it is currently in use. Through this project, I also gained valuable real-world experience in web development, networking, and both manual and automation testing in QA, which further enhanced my technical skills and prepared me for my future career in the IT industry.
